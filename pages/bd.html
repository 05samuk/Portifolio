Súmario

Já nesse tema vimos um pouco sobre os bancos de dados, lugares onde o próprio nome já diz, feito para você armazenar dados, e é muito util e importantissimo nas aplicações de hoje em dia. Podemos contar com bancos relacionais e não relacionais, usamos o SLQ uma linguagem de consulta e começamos no finalzinho do bismetre a ver o NOsql uma parte não estruturada, onde vamos explicar a baixo cada uma a relacional e não relacional, mas basicanmente alem de fazer consultas ele pode deletar, editar, excluir e criar relações entre elas podemos criar diagramas, usamos o site draw.io para criar, é importente criar antes para poder manter um código organizado. Existem padronizações vistas em aula que ajudam a deixar o banco de dados mais legivel e ocupar menos espaço a melhorar a relação e segurança.

Escolha

Lembrando um ponto muito importante a escolha de qual você irá usar depende muito da sua aplicação e como você guardará os dados. Isso é um ponto a ser definido por você ou pela sua equipe depois em conjunto e oque será melhor para aplicação e qual tipo vai atender com mais eficaz a aplicação se o SQL ou o NO SQL.

SQL

O SQL vai ser usado em bancos relacionais, vai utilzar um esquema rigido de tabelas com linhas e colunas, consultas bem estruturas usando os padrões ensinado em aula e que estão no notion para manipular os dados. Exemplos de bancos relacionais são MySQL, PostgreSQL, Oracle, SQL Server.

NO SQL

O NO SQL são os bancos relacionais temos como exemplo MongoDB, Cassandra, Redis, Couchbase, Neo4j é mais flexivel escalavel para grandes volumes muito bom para trabalhar com dados mais complexos como fotos,videos e assim por diante, Não possui esquema rígido, permitindo adição de campos sem alterar a estrutura, ele tem um formato bem parecido com um json e é muito usado em aplicações web, Big Data, IoT, análise de dados em tempo real.

Comandos principais em SQL

DDL (Data Definition Language) CREATE: Utilizado para criar novas tabelas, bancos de dados, índices ou visões.br

ALTER: Usado para modificar a estrutura de uma tabela existente.

DROP: Utilizado para remover tabelas, bancos de dados, índices ou visões.

- DML (Data Manipulation Language)

SELECT: Utilizado para consultar e recuperar dados de uma tabela.

INSERT: Usado para inserir novos registros em uma tabela.

UPDATE: Utilizado para atualizar registros existentes em uma tabela.

DELETE: Usado para remover registros de uma tabela.

- DCL (Data Control Language)

GRANT: Utilizado para conceder permissões a usuários.

REVOKE: Usado para revogar permissões concedidas a usuários.

- TCL (Transaction Control Language)

COMMIT: Utilizado para confirmar uma transação.

ROLLBACK: Usado para desfazer uma transação.

SAVEPOINT: Utilizado para criar um ponto de salvamento dentro de uma transação.



Comandos principais em NoSQL (MongoDB)

Inserção de documentos: Operação usada para adicionar novos documentos a uma coleção.

Consulta de documentos: Operação para recuperar documentos que atendem a critérios específicos de uma coleção.

Atualização de documentos: Operação para modificar documentos existentes em uma coleção.

Remoção de documentos: Operação para deletar documentos de uma coleção.

Comandos de indexação: Operações para criar ou remover índices em coleções para otimizar consultas.

Operações de agregação: Operações avançadas para processar e analisar dados em coleções, como agrupamentos e somas.
